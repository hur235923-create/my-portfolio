# JW중외제약 리디자인 — UX 케이스 스터디

**"생명이 흐르게 하다" / Pharma · Interactive Brand (고요의 심해)**  
**분석: 허강범 (Heo Gangbeom) · 2026**  

완성된 리디자인을 UX 관점에서 역설계하여, 무엇이 문제였는지 → 왜 문제인지 →
어떻게 해결했는지의 논리로 정리한다. '기존'은 국내 대형 제약사 기업 사이트의
통상적 패턴을 기준으로 한다.

## 1. 서비스 이해 (Service Understanding)

· 브랜드 목적
  - 1945년 광복과 함께 태어난 제약사. 국내 최초 수액(1959 5% 포도당)에서 시작된 헤리티지.
  - 브랜드 메시지 "생명이 흐르게 하다(Keeping Life in Flow)" — '흐름(flow)'이 핵심 은유.
  - 헤리티지 + 혁신신약(헴리브라·리바로) + AI 신약개발 플랫폼 'JWave'를 함께 전달.

· 핵심 가치
  - 생명존중 · 개척정신 · 80년 헤리티지 · 연구 기술력 · 사회적 신뢰.

· 주요 사용자 (기업 사이트 특성상 다층)
  - (A) 일반/환자·보호자: 회사가 신뢰할 만한가, 어떤 약을 만드는가.
  - (B) 투자자/구직자/파트너: 규모·성장·R&D 경쟁력(JWave)을 확인.
  - (C) 채용담당자/동료: 브랜드 전략 해석력과 인터랙티브 구현력을 평가.

· 방문 목적
  - 딱딱한 기업 정보가 아니라 '이 회사가 생명을 대하는 태도'를 신뢰감 있게 체감.
  - 헤리티지(과거)와 혁신(미래)이 하나의 흐름으로 이어짐을 이해.

## 2. 사용자 목표 분석 (User Goals)

· 가장 먼저 찾는 정보
  - "이 회사는 믿을 만한가(규모·연혁)" → "무엇을 해왔는가(헤리티지)" →
    "지금·앞으로 무엇을 하는가(R&D·제품)".

· 주요 행동 흐름
  1) Hero(심해)에서 브랜드 정서('고요·깊이') 감지
  2) Glance에서 수치(1945/80년/매출/국내최초)로 신뢰 형성
  3) Heritage 타임라인으로 헤리티지 서사 이해
  4) R&D(JWave)에서 미래 경쟁력 확인
  5) Products 세 갈래로 대표 제품 인지 → Sustainability 철학으로 잔상

· 사용자의 진짜 니즈
  - 제약사 특유의 딱딱함·정보 나열이 아닌, 정서적으로 신뢰가 쌓이는 서사.
  - 과거(수액)와 미래(AI)가 단절되지 않고 '한 흐름'으로 연결되는 이해.

## 3. 기존 UX 분석 (Baseline UX Diagnosis)

· 정보구조(IA)
  - 통상 제약 기업 사이트는 회사소개/IR/제품/채용/윤리경영이 병렬 나열된 '기업 포털'.
  - 헤리티지·비전·제품이 각 하위 페이지로 흩어져 '하나의 서사'로 읽히지 않음.

· 내비게이션
  - 다층 GNB·방대한 메뉴로 정보량이 크고, 브랜드 정서보다 '기능적 탐색'에 치우침.

· 시각적 계층
  - 보도자료·공지·배너가 균질하게 나열되어 브랜드 메시지의 위계가 약함.
  - 스톡성 이미지·표 위주라 '생명·흐름'의 정서가 시각적으로 부재.

· 콘텐츠
  - 사실 나열(설립·매출·제품명) 중심. '생명을 대하는 태도'라는 정서적 축이 약함.

· 인터랙션
  - 정적 페이지·탭 전환 위주. '흐름(flow)'이라는 핵심 은유가 움직임으로 구현되지 않음.

· 접근성
  - 인터랙티브/모션 강화 시 스크린리더·키보드·모션 민감 사용자, 한·영 병기 처리가 소홀해지기 쉬움.

· 모바일 사용성
  - 다층 패럴랙스·캔버스 파티클·배경 이미지의 성능 부담, 세로 타이포 '강(江)'의 소형 화면 처리.

## 4. 문제 발견 (Pain Points)

P1. '기업 포털'식 병렬 IA가 브랜드 메시지를 흩어 '생명을 대하는 태도'가 전달되지 않는다.
P2. 헤리티지(과거)와 R&D(미래)가 분리되어 '한 흐름'이라는 핵심 서사가 끊긴다.
P3. 사실 나열·스톡 이미지로 '생명·흐름'의 정서가 시각적으로 부재하다.
P4. '흐름(flow)'이라는 브랜드 은유를 체감할 인터랙션 장치가 없다.
P5. 딱딱한 기업 톤으로 신뢰가 '숫자'로만 쌓일 뿐 정서적 공감으로 이어지지 않는다.

## 5. 원인 분석 (Why)

P1 ← 부서·기능 단위(회사/IR/제품/채용)로 IA를 구성해 브랜드 서사를 통합하지 못함.
P2 ← 과거와 미래를 별도 메뉴로 다루고 '흐름'이라는 연결 개념을 IA로 승격하지 않음.
P3 ← 이미지를 정보의 삽화로만 사용, 브랜드 은유(물·심해)를 시각 시스템으로 설계하지 않음.
P4 ← 모션을 장식으로 인식. 핵심 은유(flow)를 관통하는 시각 장치가 부재.
P5 ← 신뢰를 정량 지표로만 환원하고 '생명존중'의 정서적 서사를 조형화하지 않음.

## 6. UX 문제 정의 (Problem Statement)

· 사용자는 "이 제약사가 생명을 대하는 태도와, 과거(수액)에서 미래(AI)로 이어지는 흐름"을
  신뢰감 있게 이해하려 하지만, 기업 포털식 병렬 IA와 사실 나열 탓에 브랜드 메시지가 흩어져
  정서적 신뢰에 도달하지 못한다.
· 사용자는 긴 브랜드 스토리에서 '흐름'을 체감하고 싶지만, 이를 시각·인터랙션으로 매개하는
  장치가 없어 몰입이 약하다.

## 7. 프로젝트 목표 (Goal)

G1. 기업 포털을 '고요의 심해' 단일 서사로 재편해 브랜드 메시지를 하나로 모은다.
G2. 과거→현재→미래를 '흐름(flow)'이라는 하나의 축으로 연결한다.
G3. '물·심해·흐름' 은유를 시각·인터랙션 시스템으로 구현해 정서적 신뢰를 만든다.
G4. 한·영 바이링구얼 프리미엄 경험을 제공하되 성능·접근성을 지킨다.

## 8. UX 전략 (Strategy)

S1. 서사 통합 IA: Hero → Glance(현황) → Heritage(과거) → R&D(미래·JWave)
    → Products(제품) → Sustainability(철학)의 '깊이로 내려가는' 단일 스토리로 재편.
S2. 관통하는 은유 장치: 좌측을 세로로 관통하는 타이포 '강(江)'(river) + 진행 게이지가
    페이지 전체에 '흐름'을 상시 각인(P4 해결, G3).
S3. 물의 시각 시스템: 심해·잉크·바다·기포 실사 이미지 + 다층 패럴랙스(data-depth)로
    '깊이'를 조형화. Hero는 비대칭 다층 레이어로 심해감 연출(P3 해결).
S4. 흐름의 연결: Heritage 세로 연도 타임라인(1945→1959→1969→Today)이
    R&D의 "물의 흐름이 데이터의 흐름으로"로 자연스럽게 이어져 과거·미래를 봉합(P2 해결, G2).
S5. 정서적 신뢰: Glance 카운트업 수치를 "숫자가 아니라 흐름의 깊이"로 프레이밍하고,
    Sustainability에서 "생명이 흐르는 한 우리의 흐름도 멈추지 않는다"로 정서를 클로징(P5 해결).
S6. 접근성·성능: skip-link, 캔버스/장식 aria-hidden, 한·영 병기, 시맨틱 섹션,
    prefers-reduced-motion·저사양 대비 고려.

## 9. UI 적용 (Design Solution — 실제 구현 매핑)

· 톤&매너: 딥 블루/블랙 심해 팔레트 + Noto Sans KR 본문 + JetBrains Mono(데이터/라벨)로
  '고요·연구·신뢰'의 정서. 세로 라벨(vlabel)·모노 폰트로 '연구소' 감성 은유.
· 프리로더(#loader): 한 방울(drop)이 심연으로 떨어지며 숫자 카운트 → '한 방울에서 시작된
  흐름'이라는 브랜드 기원을 첫 순간에 각인.
· 세로 타이포 강(#river): "생명이 흐르게 하다 · KEEPING LIFE IN FLOW · SINCE 1945" +
  riverFill 진행 게이지가 좌측을 관통 → '흐름' 은유를 상시 체감(P4 해결, G3).
· Hero(#hero): deep.jpg 심해 배경의 다층 패럴랙스(img/veil/glow, data-depth) + 비대칭 타이포
  "한 방울이 생명을 잇다" + 한·영 병기로 정서·격을 즉시 형성(P3 해결, G4).
· Glance(#glance): 카운트업 지표(1945/80년/7194억/국내최초)를 "숫자가 아니라 흐름의 깊이"로
  프레이밍 → 신뢰를 정서로 전환(P5 해결). 잉크 확산 이미지로 '흐름' 시각화.
· Heritage(#heritage): 세로 연도 타임라인(drop, 1945→1959→1969→Today) + 패럴랙스 미디어로
  헤리티지를 '아래로 흐르는' 서사로 구현(G2).
· R&D(#rnd): 연구 실사 + <canvas> 데이터 파티클(field) + "물의 흐름이 데이터의 흐름으로"로
  과거(수액)와 미래(AI JWave)를 은유적으로 봉합(P2 해결).
· Products(#products): 위너프/리바로/헴리브라를 '세 갈래의 흐름'으로 겹치는 어긋난 블록에 배치,
  각 제품에 '흐름' 카피 부여.
· Sustainability(#sustain): "생명이 흐르는 한 우리의 흐름도 멈추지 않는다" 대형 인용으로
  브랜드 철학을 정서적 잔상으로 클로징.
· 접근성: skip-link(#glance), 섹션 aria-labelledby, 캔버스/river/장식 aria-hidden,
  의미 있는 alt, 한·영 병기 텍스트.

## 10. 기대 효과 (Expected Outcome)

· 기업 포털이 '고요의 심해' 단일 서사로 모여 브랜드 메시지가 명확하고 정서적으로 각인된다.
· 세로 타이포 강·패럴랙스가 '흐름' 은유를 상시 체감시켜 몰입과 기억도를 높인다.
· 헤리티지 타임라인 → R&D로 이어지는 구성으로 과거·미래가 '한 흐름'으로 이해된다.
· 카운트업 수치의 정서적 프레이밍으로 신뢰가 숫자를 넘어 공감으로 확장된다.
· 한·영 병기·접근성 배려로 폭넓은 사용자에게 프리미엄 경험을 균질하게 제공한다.

[검증 메모] IA·콘텐츠·인터랙션 구조를 코드 레벨에서 확인해 작성했다.
다층 패럴랙스·캔버스 데이터 파티클의 실제 60fps 성능과 모바일 발열/배터리,
세로 타이포 '강'의 소형 화면 표시·가로 오버플로우는 브라우저 실측(데스크톱+모바일)으로
별도 확인 권장.

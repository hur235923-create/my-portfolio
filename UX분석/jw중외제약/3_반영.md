# JW중외제약 리디자인 — UX 케이스 스터디 ③ 반영 (Implementation)

**"생명이 흐르게 하다" / Pharma · Interactive Brand (고요의 심해)**
**변경 파일: works/jw중외제약/ index.html · styles.css · script.js**
**검증: 로컬 서버 + Playwright (데스크톱 1440 / 모바일 390)**
**작성: 허강범 · 2026**

[② 개선안](2_개선안.md)의 설계를 실제 화면·코드로 구현한 결과다.
[① 분석](1_분석.md)의 문제 **P1~P5**가 화면에서 어떻게 해결됐는지(§1)와,
완성 후 추가로 잡은 결함 보강의 반영(§2)을 함께 기록한다.

## 1. 리디자인 구현 — 문제 해결 매핑 (P1~P5)

· 톤&매너: 딥 블루/블랙 심해 팔레트 + Noto Sans KR 본문 + JetBrains Mono(데이터/라벨)로
  '고요·연구·신뢰'의 정서. 세로 라벨(vlabel)·모노 폰트로 '연구소' 감성 은유.
· 프리로더(#loader): 한 방울(drop)이 심연으로 떨어지며 숫자 카운트 → '한 방울에서 시작된
  흐름'이라는 브랜드 기원을 첫 순간에 각인.
· 세로 타이포 강(#river): "생명이 흐르게 하다 · KEEPING LIFE IN FLOW · SINCE 1945" +
  riverFill 진행 게이지가 좌측을 관통 → '흐름' 은유를 상시 체감. (P4 해결)
· Hero(#hero): deep.jpg 심해 배경의 다층 패럴랙스(img/veil/glow) + 비대칭 타이포
  "한 방울이 생명을 잇다" + 한·영 병기로 정서·격을 즉시 형성. (P3 해결)
· Glance(#glance): 카운트업 지표(1945/80년/7194억/국내최초)를 "숫자가 아니라 흐름의 깊이"로
  프레이밍 → 신뢰를 정서로 전환. (P5 해결)
· Heritage(#heritage): 세로 연도 타임라인(1945→1959→1969→Today) + 패럴랙스 미디어로
  헤리티지를 '아래로 흐르는' 서사로 구현. (P1·P2)
· R&D(#rnd): 연구 실사 + <canvas> 데이터 파티클 + "물의 흐름이 데이터의 흐름으로"로
  과거(수액)와 미래(AI JWave)를 은유적으로 봉합. (P2 해결)
· Products(#products): 위너프/리바로/헴리브라를 '세 갈래의 흐름'으로 겹치는 어긋난 블록에 배치.
· Sustainability(#sustain): "생명이 흐르는 한 우리의 흐름도 멈추지 않는다" 대형 인용으로
  브랜드 철학을 정서적 잔상으로 클로징.
· 접근성: skip-link(#glance), 섹션 aria-labelledby, 캔버스/river/장식 aria-hidden, 의미 있는 alt, 한·영 병기.

### 기대 효과 (Expected Outcome)

· 기업 포털이 '고요의 심해' 단일 서사로 모여 브랜드 메시지가 명확·정서적으로 각인된다. (P1)
· 세로 타이포 강·패럴랙스가 '흐름' 은유를 상시 체감시켜 몰입·기억도를 높인다. (P4)
· 헤리티지 타임라인 → R&D로 이어져 과거·미래가 '한 흐름'으로 이해된다. (P2)
· 카운트업 수치의 정서적 프레이밍으로 신뢰가 숫자를 넘어 공감으로 확장된다. (P5)
· 한·영 병기·접근성 배려로 폭넓은 사용자에게 프리미엄 경험을 균질하게 제공한다.

## 2. 추가 결함 보강 — 반영 (P1~P6, 구현 레벨)

[② 개선안 §3]의 2차 개선을 실제로 반영·검증한 기록이다. (여기 P1~P6은 위 브랜드 문제와
라벨만 겹치는 구현 레벨 결함이다.) 각 변경은 [발견한 문제 → 개선 목표 → 디자인 해결 방법
→ 기대 효과 → 검증]으로 기술한다.

## P1 — 모바일 내비게이션 드로어

· 발견한 문제: 모바일(≤760)에서 메뉴를 대체 없이 display:none → 섹션 이동 수단 전무.
· 개선 목표: 모바일에서도 4개 섹션·문의 탐색 경로 회복.
· 디자인 해결: 햄버거 → 심해 풀스크린 드로어(섹션+부제, cyan '문의하기' CTA).
  aria-expanded/controls, 포커스 트랩, ESC, 스크롤 락(프리로더 패턴 재사용), 포커스 복귀.
  reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 태스크 완료 경로 확보, 이탈 감소.
· 검증: 버거 노출·메뉴 숨김·드로어 열림·focusInside=true·ESC 닫힘 통과.

## P2 — 모바일 '흐름' 진행 게이지 (river 대체)

· 발견한 문제: 세로 타이포 강이 모바일에서 소멸 → 브랜드 은유·위치감 상실.
· 개선 목표: 모바일에서도 '흐름' 은유·스크롤 위치감 유지.
· 디자인 해결: 헤더 하단 얇은 진행 바(cyan→blue). 기존 riverFill 계산값 재사용, scaleX만 반영.
· 기대 효과: 브랜드 일관성 + wayfinding 회복.
· 검증: 모바일 게이지 노출, 중간 스크롤 scaleX 0.55 연동 확인.

## P3 — 태스크 종착지 / 세그먼트 CTA

· 발견한 문제: 대표 CTA가 #top으로 되돌아가는 막다른 루프. 행동 종착지 부재.
· 개선 목표: 다중 사용자군(환자·HCP·투자자·구직자)에 실제 행동 경로 제공.
· 디자인 해결: 문의하기(mailto)·제품 살펴보기·인재영입/투자정보(준비 중 명시).
  푸터 '둘러보기/바로가기' 2그룹. 미완 링크는 aria-disabled로 오인 방지.
· 기대 효과: 탐색 완결성·전환 경로·신뢰 상승.
· 검증: CTA·2그룹·비활성 표시 정상, 비활성 링크 클릭 무력화.

## P4 — 제품 정보 1-스텝 심화 (아코디언)

· 발견한 문제: 제품 3종이 감성 카피뿐, '자세히' 어포던스 0.
· 개선 목표: 감성 인상 유지 + 최소 사실 정보 제공.
· 디자인 해결: 카드별 접근 가능 disclosure(aria-expanded, +→−) — 성분·적응증·특징.
  키보드 지원, reduced-motion 시 애니메이션 제거, 열림 시 ScrollTrigger.refresh().
· 기대 효과: 환자·보호자·HCP의 이해·신뢰 심화.
· 검증: false→true 토글·패널 노출 확인.

## P5 — 신뢰·안전(YMYL) 고지

· 발견한 문제: 의약품 정보인데 전문가 상담 고지·정보 성격 안내 부재.
· 개선 목표: 제약 도메인 신뢰(E-E-A-T)·오용 리스크 저감.
· 디자인 해결: 제품 섹션 하단 + 푸터에 "의학적 판단은 의료 전문가와 상담" 고지문.
· 기대 효과: 신뢰성 보강, 책임 있는 정보 제공.

## P6 — 접근성 폴리시

· 발견한 문제: hero__mark 중복 낭독 / en 저대비 소지 / 카운트업 SR 불안정 / nav 활성 표시 없음.
· 개선 목표: 스크린리더·저시력·wayfinding 접근성 향상.
· 디자인 해결: ①hero__mark aria-hidden ②.en --mute→--ink-s, .en--top→--deep, .flow__no→--ink-s
  ③애니메이션 span aria-hidden + dt aria-label 확정값 ④스크롤 시 aria-current + 밑줄 유지.
· 기대 효과: 폭넓은 사용자에게 균질한 프리미엄 경험.
· 검증: 스크롤 시 활성 nav(#products/#sustain) 정확히 갱신 확인.

· 데스크톱: 아코디언·활성 nav 정상 / 모바일: 드로어·포커스 트랩·ESC·게이지 정상
· 컨셉(고요의 심해·세로 강·다층 패럴랙스) 보존, 결함만 국소 보강. 신규 라이브러리 0.

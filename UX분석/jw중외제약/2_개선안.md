# JW중외제약 리디자인 — UX 케이스 스터디 ② 개선안 (Design Solution)

**"생명이 흐르게 하다" / Pharma · Interactive Brand (고요의 심해)**
**설계: 허강범 (Heo Gangbeom) · 2026**

[① 분석](1_분석.md)에서 도출한 문제 **P1~P5**를 어떻게 풀지 설계하는 단계다.
브랜드 분석(①) → **이 설계(②)** → 실제 구현·검증([③ 반영](3_반영.md))으로 이어진다.
각 목표·전략은 어떤 문제(P#)를 겨냥하는지 괄호로 표기한다.

## 1. 프로젝트 목표 (Goal)

G1. 기업 포털을 '고요의 심해' 단일 서사로 재편해 브랜드 메시지를 하나로 모은다. (→ P1)
G2. 과거→현재→미래를 '흐름(flow)'이라는 하나의 축으로 연결한다. (→ P2)
G3. '물·심해·흐름' 은유를 시각·인터랙션 시스템으로 구현해 정서적 신뢰를 만든다. (→ P3·P4)
G4. 한·영 바이링구얼 프리미엄 경험을 제공하되 성능·접근성을 지킨다. (→ P5)

## 2. UX 전략 (Strategy)

S1. 서사 통합 IA: Hero → Glance(현황) → Heritage(과거) → R&D(미래·JWave)
    → Products(제품) → Sustainability(철학)의 '깊이로 내려가는' 단일 스토리로 재편. (→ P1)
S2. 관통하는 은유 장치: 좌측을 세로로 관통하는 타이포 '강(江)'(river) + 진행 게이지가
    페이지 전체에 '흐름'을 상시 각인. (→ P4·G3)
S3. 물의 시각 시스템: 심해·잉크·바다·기포 실사 이미지 + 다층 패럴랙스(data-depth)로
    '깊이'를 조형화. Hero는 비대칭 다층 레이어로 심해감 연출. (→ P3)
S4. 흐름의 연결: Heritage 세로 연도 타임라인(1945→1959→1969→Today)이
    R&D의 "물의 흐름이 데이터의 흐름으로"로 이어져 과거·미래를 봉합. (→ P2)
S5. 정서적 신뢰: Glance 카운트업 수치를 "숫자가 아니라 흐름의 깊이"로 프레이밍하고,
    Sustainability에서 "생명이 흐르는 한 우리의 흐름도 멈추지 않는다"로 정서를 클로징. (→ P5)
S6. 접근성·성능: skip-link, 캔버스/장식 aria-hidden, 한·영 병기, 시맨틱 섹션,
    prefers-reduced-motion·저사양 대비 고려.

→ 위 설계를 실제 화면·코드로 구현한 결과와 검증은 [③ 3_반영.md](3_반영.md)에서 다룬다.

## 3. 추가 결함 보강 (2차 개선)

위 리디자인을 완성한 뒤 **코드에서 추가로 발견한 실재 결함**을 정밀 보강하는 설계다.
아래 라벨 **P1~P6은 ①분석의 브랜드 문제 P1~P5와 라벨만 겹칠 뿐 다른 층위**
(구현 레벨 결함 — 모바일 내비·흐름 게이지·CTA·아코디언·YMYL·접근성)다. 예술적 컨셉(고요의
심해·세로 타이포 강·다층 패럴랙스)은 보존한 채 결함만 보강하며, 각 항목은
[문제 → 근거(파일:라인) → 개선 설계 → 기대효과 → 구현 범위] 순으로 기술한다.

## 0. 설계 원칙 (Guiding Principles)

1) 컨셉 보존: 비주얼 시스템·팔레트(:root 토큰)·모션 언어를 유지한다.
2) 접근성 기본 탑재: 신규 인터랙션은 키보드·스크린리더·포커스 관리를 처음부터 포함.
3) 성능 가드레일: 캔버스/blur 등 무거운 요소를 추가하지 않고, 기존 스크롤 로직·
   토큰을 재사용해 코드 중복과 부하를 최소화.
4) 점진 개선: 원본 구조를 뒤엎지 않고 결함만 국소적으로 교정.
5) reduced-motion·모바일 우선: 모든 신규 모션은 prefers-reduced-motion에서 축소.

## 개선 항목 우선순위 요약

P1 🔴 모바일 내비게이션 복원 (치명)
P2 🔴 모바일 '흐름' 은유 연속성 (치명)
P3 🔴 태스크 종착지 / 세그먼트 CTA (치명)
P4 🟡 제품 정보 1-스텝 심화 (경량)
P5 🟡 신뢰·안전(YMYL) 요소
P6 🟡 접근성 폴리시

## P1 — 모바일 내비게이션 복원 🔴

· 문제
  모바일(≤760px)에서 주요 섹션 메뉴가 대체 수단 없이 사라져, 사용자가
  Heritage·R&D·Products·Sustainability로 이동할 방법이 전혀 없다.

· 근거
  - styles.css:272–276  @media (max-width:760px){ .nav__menu{ display:none } }
    → 햄버거/드로어 등 대체 없이 메뉴만 은닉.
  - 데스크톱 nav(index.html:40–45)에만 섹션 앵커가 존재.

· 개선 설계
  - 우상단에 햄버거 버튼 신설(≤760px에서만 노출), 클릭 시 '풀스크린 심해 드로어'
    오버레이로 4개 섹션 + 브랜드 링크 제공.
  - 톤: 배경 var(--abyss)/(--abyss2), 강조 var(--cyan) — 기존 심해 팔레트 재사용.
  - 접근성:
    · 버튼 <button aria-label="메뉴 열기" aria-expanded aria-controls="mNav">
    · 열림 시 aria-expanded=true, 첫 링크로 포커스 이동, ESC로 닫기.
    · 포커스 트랩(드로어 내부 Tab 순환), 닫으면 햄버거 버튼으로 포커스 복귀.
    · 열릴 때 스크롤 락 — 기존 프리로더의 jw_lenis.stop()+overflow:hidden 패턴 재사용
      (script.js:68–69), 닫힐 때 jw_lenis.start().
  - 모션: 슬라이드/페이드 0.4s var(--ease). prefers-reduced-motion에서는 즉시 페이드만.
  - 링크 클릭 시 기존 data-link 앵커 로직(script.js:36–44)으로 부드러운 스크롤 후 닫기.

· 기대효과
  모바일 다수 사용자의 섹션 접근성 회복 → 투자자·구직자·환자 태스크 완료 경로 확보.

· 구현 범위
  index.html(헤더에 버튼+드로어 마크업) / styles.css(드로어·햄버거 스타일, ≤760 노출) /
  script.js(토글·포커스 트랩·스크롤 락). 신규 라이브러리 없음.

## P2 — 모바일 '흐름' 은유 연속성 🔴

· 문제
  컨셉의 핵심 장치인 세로 타이포 강(river)이 모바일에서 완전히 사라져,
  '흐름(flow)' 은유와 스크롤 위치감(wayfinding)이 동시에 소실된다.

· 근거
  - styles.css:89  @media (max-width:760px){ .river{ display:none } }
  - 진행 게이지 로직은 script.js:22–27 (riverFill scaleY = scroll/max)에 이미 존재.

· 개선 설계
  - 모바일에서 river 대신 '상단 얇은 진행 게이지 바'(높이 2px, cyan→blue 그라디언트)를
    헤더 하단 라인에 노출.
  - 기존 riverFill 스크롤 계산값(p)을 그대로 재사용해 별도 로직/부하 없이 폭(scaleX)만 반영.
  - 데스크톱은 기존 river 유지, 모바일만 게이지 바로 대체(둘 다 같은 진행값 공유).

· 기대효과
  모바일에서도 '흐름' 은유를 최소한 유지하고 스크롤 진척을 시각화 → 브랜드 일관성 +
  위치감 회복.

· 구현 범위
  index.html(게이지 바 요소 또는 nav 하단 활용) / styles.css(≤760 노출) /
  script.js(기존 스크롤 핸들러에서 게이지 바 scaleX 갱신 1줄 추가).

## P3 — 태스크 종착지 / 세그먼트 CTA 🔴

· 문제
  제약사는 사용자군(환자·보호자 / 의료전문가(HCP) / 투자자 / 구직자)이 다양한데,
  실제 행동으로 이어지는 종착지가 없다. 대표 CTA가 맨 위로 되돌아가는 막다른 루프다.

· 근거
  - index.html:216–219  <a class="foot__make" href="#top" …>다음 흐름에 함께</a>
    → 클릭해도 페이지 최상단으로 스크롤될 뿐, 어떤 행동도 완료할 수 없음.
  - 제품 카드(index.html:174–202)에 상세·문의 링크 부재.

· 개선 설계
  - 대표 CTA 재정의: "다음 흐름에 함께"를 실제 목적지로 연결.
    · 최소안: 문의 경로(문의 앵커 섹션 또는 mailto), 그리고 IR·채용은 외부/placeholder 링크.
  - 푸터에 '사용자군 진입 링크' 블록 명시:
    · 제품 정보 / R&D·파이프라인 / 지속가능경영 / 채용 / 문의(IR·PR)
    · 실제 페이지가 없는 항목은 접근 가능한 placeholder(#) + 상태 안내(예: aria-disabled 또는
      '준비 중' 라벨)로 처리해 '막다른 링크'로 오인되지 않게 한다.
  - 톤: 기존 foot 스타일(mono 라벨, cyan hover) 재사용.

· 기대효과
  다중 사용자군에게 명확한 행동 종착지를 제공하고, 되돌이 루프를 제거 → 전환·탐색 완결성↑.

· 구현 범위
  index.html(푸터 CTA/링크 블록, 필요 시 문의 앵커) / styles.css(경미) — 새 페이지 신설은
  범위 밖(placeholder 링크로 표현).

## P4 — 제품 정보 1-스텝 심화 (경량) 🟡

· 문제
  위너프·리바로·헴리브라가 감성 카피만 있고, 무엇에 쓰는 약인지 한 단계 더 알 방법이 없다.

· 근거
  - index.html:174–202  각 flow 카드는 tag(적응증 한 줄)+감성 desc뿐, 상세 없음.

· 개선 설계
  - 각 flow 카드에 접근 가능한 disclosure(아코디언) 추가:
    · <button aria-expanded aria-controls> "자세히" → 성분/적응증/특징 요약 3~4줄 노출.
    · 키보드(Enter/Space) 지원, 열림 상태 aria-expanded 반영.
  - 감성 톤(카피)은 유지하고 정보를 '옵션'으로 한 겹 얹어 과잉 노출을 피함.
  - 모션: 높이 전개는 CSS grid-rows 트랜지션 또는 GSAP, reduced-motion에서 즉시 표시.

· 기대효과
  환자·보호자·HCP가 감성 인상 이후 최소한의 사실 정보를 얻어 신뢰·이해가 깊어짐.

· 구현 범위
  index.html(카드별 disclosure 마크업+요약 텍스트) / styles.css(아코디언) /
  script.js(토글). 실제 제품 상세 페이지는 범위 밖.

## P5 — 신뢰·안전(YMYL) 요소 🟡

· 문제
  의약품 관련 정보를 다루는 제약 사이트(YMYL: Your Money Your Life)인데,
  전문가 상담 고지·정보 성격 안내 등 신뢰 신호가 없다.

· 근거
  - 제품 섹션(index.html:169–203)·푸터(215–230)에 안전/면책 문구 부재.
  - 이미지 출처·브랜드 자산 고지는 이미 존재(index.html:228) — 여기에 안전 문구를 보강.

· 개선 설계
  - 제품 섹션 하단 또는 푸터에 고지문 추가:
    "본 사이트의 제품 정보는 일반적 안내이며, 복용·처방 등 의학적 판단은 반드시
     의료 전문가와 상담하시기 바랍니다."
  - 시각: mono 소형 캡션(기존 foot__copy 톤), 과하지 않게.

· 기대효과
  제약 도메인의 전문성·신뢰성(E-E-A-T) 보강, 오해·오용 리스크 저감.

· 구현 범위
  index.html(고지 문구) / styles.css(경미).

## P6 — 접근성 폴리시 🟡

· P6-1 장식 텍스트 중복 낭독 제거
  - 문제/근거: index.html:60 <p class="hero__mark">JW중외제약</p> 이 aria-hidden 아님 →
    스크린리더가 nav 브랜드와 별개로 대형 장식 브랜드명을 중복 낭독.
  - 설계: hero__mark에 aria-hidden="true" 추가(순수 장식).

· P6-2 영문 캡션 대비 상향
  - 문제/근거: .en 캡션이 var(--mute #8093a2) 소형 mono(styles.css:246) → 밝은 배경(--foam)
    위 소형 텍스트 대비가 WCAG AA(4.5:1) 미달 소지.
  - 설계: .en 색을 더 진한/밝은 토큰으로 조정하거나 최소 폰트 크기·두께 상향해 AA 충족.
    (라이트 섹션은 --ink-s 계열, 다크 섹션은 기존 cyan/blue 유지 검토)

· P6-3 카운트업 수치의 스크린리더 처리
  - 문제/근거: [data-count-to] 수치(script.js:132–141)가 애니메이션 중 텍스트가 계속 바뀜 →
    스크린리더가 중간값을 읽거나 최종값 인지가 불안정.
  - 설계: 애니메이션 노출용 span은 aria-hidden, 최종값을 시각적 대체(visually-hidden) 또는
    dt에 aria-label로 확정값 제공. reduced-motion에서는 이미 최종값 표기(script.js:135).

· P6-4 데스크톱 nav 현재 섹션 활성 표시
  - 문제/근거: nav 링크에 현재 위치 표시가 없어(강 게이지는 전체 진행만) 롱스크롤 wayfinding 약함.
  - 설계: ScrollTrigger로 각 섹션 진입 시 해당 nav 링크에 aria-current="location" + 밑줄 유지
    (styles.css:106 밑줄 스타일 재사용). reduced-motion 무관(상태 표시일 뿐).

· 구현 범위
  index.html(aria 속성) / styles.css(.en 대비, active 상태) / script.js(active 토글,
  카운트업 aria 처리).

## 반응형 · 성능 가드레일

· 반응형 3분기 유지: 데스크톱 / 태블릿(≤960) / 모바일(≤760). 신규 요소도 동일 분기 준수.
· 성능: 신규 캔버스·backdrop-filter 추가 금지. 진행 게이지는 기존 스크롤 값 재사용.
· 모션: 모든 신규 애니메이션은 prefers-reduced-motion:reduce에서 축소/제거.
· 코드: :root 토큰·var(--ease)·기존 lenis/ScrollTrigger 패턴 재사용(중복 최소화).

## 검증 체크리스트 (구현 단계에서 실행)

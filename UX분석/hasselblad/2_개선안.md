# HASSELBLAD 리디자인 — UX 케이스 스터디 ② 개선안 (Design Solution)

**"Legacy in Every Frame" / Heritage Camera · Dark Luxury**
**설계: 허강범 (Heo Gangbeom) · 2026**

[① 분석](1_분석.md)에서 도출한 문제 **P1~P5**를 어떻게 풀지 설계하는 단계다.
브랜드 분석(①) → **이 설계(②)** → 실제 구현·검증([③ 반영](3_반영.md))으로 이어진다.
각 목표·전략은 어떤 문제(P#)를 겨냥하는지 괄호로 표기한다.

## 1. 프로젝트 목표 (Goal)

G1. 진입 3초 안에 '헤리티지 · 다크 럭셔리'라는 브랜드 격을 체감시킨다. (→ P1·P5)
G2. 제품을 'X · V · H System'이라는 하나의 정돈된 세계관으로 재구조화한다. (→ P2)
G3. 결과물(작례)을 콘텐츠의 주인공으로 승격해 시각적 설득력을 극대화한다. (→ P3)
G4. '정밀·빛'이라는 브랜드 속성을 인터랙션 언어로 번역하되 60fps·접근성을 지킨다. (→ P4)

## 2. UX 전략 (Strategy)

S1. 서사 우선 IA: Hero → About(헤리티지) → Products(시스템) → Inspiration → Ambassadors
    → Discover(갤러리)의 '감정→구조→증거' 스토리라인으로 단일 페이지를 재편. (→ P1)
S2. 절제된 내비게이션: About/Products/Inspiration/Discover 4개 앵커 + EST.1941 시그니처.
    분산된 메가메뉴를 제거해 프리미엄의 '여백'을 확보. (→ P1·P2)
S3. 제품의 시스템화: 가로 핀(pin) 쇼케이스로 X→V→H→Lens를 '한 축의 흐름'으로 제시,
    번호(01–04)로 시스템 계층을 시각화. (→ P2)
S4. 결과물 주인공화: 흩어진 작례를 Inspiration 스토리 + Discover 매트릭스 갤러리 + 라이트박스
    확대로 '하나의 갤러리'에 응집. (→ P3)
S5. 브랜드 속성의 모션 번역: (→ P4·P5)
    - 시네마틱 프리로더(로딩=막을 올리는 연출)로 첫인상 설계
    - 마우스 반응 WebGL 셰이더 히어로로 '빛'을 체감
    - 헤딩 클립 리빌·키네틱 마퀴로 '정밀·리듬' 표현
    - 커스텀 커서·마그네틱 링크로 정교함의 촉각 은유
S6. 성능·접근성 병행: skip-link, aria 라벨, 대체텍스트, prefers-reduced-motion 고려.

→ 위 설계를 실제 화면·코드로 구현한 결과와 검증은 [③ 3_반영.md](3_반영.md)에서 다룬다.

## 3. 추가 결함 보강 (2차 개선)

위 리디자인을 완성한 뒤, **코드에서 추가로 발견한 실재 결함**을 보강하는 설계다.
P1~P5(브랜드 분석에서 도출한 서사·구조 문제)와는 층위가 다른 **구현 레벨 품질 이슈**
(모바일 내비 소멸·죽은 링크·모달 접근성 등)이며, 컨셉(dark luxury·프리로더·WebGL 히어로·
가로 핀 쇼케이스·라이트박스)은 보존한 채 결함만 보강한다.

우선순위
  HP1 🔴 모바일 내비게이션 (nav__menu 소멸 대체)
  HP2 🟡 READ MORE 죽은 링크(#) 처리
  HP3 🟡 라이트박스 포커스 트랩 보강
  HP4 🟡 데스크톱 nav 현재 섹션 활성 표시

(참고) 상단 스크롤 진행바(.scroll-progress)는 모바일에서도 이미 노출되므로 별도 게이지 불필요.

### HP1 — 모바일 내비게이션 🔴

· 문제: ≤720px에서 nav 메뉴를 대체 없이 숨김 → 모바일에 섹션 이동 수단 전무.
  근거: styles.css @media(max-width:720px){ .nav__menu{ display:none } }.
· 개선 목표: 모바일에서 About·Products·Inspiration·Discover 탐색 회복.
· 디자인 해결: 우상단 햄버거 → dark luxury 풀스크린 드로어(섹션+부제, Playfair 헤딩).
  aria-expanded/controls, 포커스 트랩, ESC, 스크롤 락(hb_lenis), 포커스 복귀.
  reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.

### HP2 — READ MORE 죽은 링크 🟡

· 문제: 다수 "READ MORE"(.link-more)가 href="#" → 클릭 시 페이지 최상단으로 점프.
  근거: index.html 내 6곳(About·제품 카드 4·Inspiration 피처)의 a.link-more href="#".
· 개선 목표: 오작동(상단 점프) 제거, 상세 페이지 부재를 정직하게 처리.
· 디자인 해결: 해당 링크에 aria-disabled="true" + 클릭 무력화(preventDefault).
  상세 콘텐츠(제품 상세 등) 확보 시 실제 링크로 승격.
· 기대 효과: 잘못된 상호작용 제거, 사용자 혼란·신뢰 저하 방지.

### HP3 — 라이트박스 포커스 트랩 🟡

· 문제: 라이트박스가 열리면 닫기 버튼에 포커스가 가지만, Tab 시 포커스가 뒤 페이지로
  빠져나가 모달 접근성 원칙(focus trap)을 완전히 만족하지 못함.
  근거: script.js 라이트박스 블록에 Escape·backdrop·focus 복귀는 있으나 Tab 트랩 없음.
· 개선 목표: 모달 열림 동안 포커스를 모달 내부로 가둔다.
· 디자인 해결: 라이트박스 open 상태에서 Tab을 닫기 버튼으로 고정(포커스 이탈 방지).
· 기대 효과: 스크린리더/키보드 사용자의 모달 조작 일관성 확보.

### HP4 — 데스크톱 nav 현재 섹션 활성 표시 🟡

· 문제: nav 링크에 현재 섹션 표시가 없어 롱스크롤 wayfinding이 약함.
· 개선 목표: 현재 위치를 nav에 표시.
· 디자인 해결: 스크롤 시 활성 섹션 링크에 aria-current="location" + 밑줄 유지
  (기존 밑줄 스타일 재사용).
· 기대 효과: 데스크톱 wayfinding 향상.

검증 체크리스트: 콘솔/네트워크 0오류, 가로 오버플로우 0, 드로어 키보드/포커스 트랩/ESC,
READ MORE 무력화, 라이트박스 트랩, nav aria-current, reduced-motion, 데스크톱+모바일 스크린샷.
범위 밖: 제품 상세 페이지 신설(placeholder 처리).

## HP1 — 모바일 내비게이션 🔴

· 문제: ≤720px에서 nav 메뉴를 대체 없이 숨김 → 모바일에 섹션 이동 수단 전무.
  근거: styles.css:370–371 @media(max-width:720px){ .nav__menu{ display:none } }.
· 개선 목표: 모바일에서 About·Products·Inspiration·Discover 탐색 회복.
· 디자인 해결: 우상단 햄버거 → dark luxury 풀스크린 드로어(섹션+부제, Playfair 헤딩).
  aria-expanded/controls, 포커스 트랩, ESC, 스크롤 락(hb_lenis), 포커스 복귀.
  reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.

## HP2 — READ MORE 죽은 링크 🟡

· 문제: 다수 "READ MORE"(.link-more)가 href="#" → 클릭 시 페이지 최상단으로 점프.
  근거: index.html 내 6곳(About·제품 카드 4·Inspiration 피처)의 a.link-more href="#".
· 개선 목표: 오작동(상단 점프) 제거, 상세 페이지 부재를 정직하게 처리.
· 디자인 해결: 해당 링크에 aria-disabled="true" + 클릭 무력화(preventDefault).
  상세 콘텐츠(제품 상세 등) 확보 시 실제 링크로 승격.
· 기대 효과: 잘못된 상호작용 제거, 사용자 혼란·신뢰 저하 방지.

## HP3 — 라이트박스 포커스 트랩 🟡

· 문제: 라이트박스가 열리면 닫기 버튼에 포커스가 가지만, Tab 시 포커스가 뒤 페이지로
  빠져나가 모달 접근성 원칙(focus trap)을 완전히 만족하지 못함.
  근거: script.js 라이트박스 블록에 Escape·backdrop·focus 복귀는 있으나 Tab 트랩 없음.
· 개선 목표: 모달 열림 동안 포커스를 모달 내부로 가둔다.
· 디자인 해결: 라이트박스 open 상태에서 Tab을 닫기 버튼으로 고정(포커스 이탈 방지).
· 기대 효과: 스크린리더/키보드 사용자의 모달 조작 일관성 확보.

## HP4 — 데스크톱 nav 현재 섹션 활성 표시 🟡

· 문제: nav 링크에 현재 섹션 표시가 없어 롱스크롤 wayfinding이 약함.
· 개선 목표: 현재 위치를 nav에 표시.
· 디자인 해결: 스크롤 시 활성 섹션 링크에 aria-current="location" + 밑줄 유지
  (기존 밑줄 스타일 재사용).
· 기대 효과: 데스크톱 wayfinding 향상.

검증 체크리스트: 콘솔/네트워크 0오류, 가로 오버플로우 0, 드로어 키보드/포커스 트랩/ESC,
READ MORE 무력화, 라이트박스 트랩, nav aria-current, reduced-motion, 데스크톱+모바일 스크린샷.
범위 밖: 제품 상세 페이지 신설(placeholder 처리).

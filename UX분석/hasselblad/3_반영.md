## 1. 리디자인 구현 — 문제 해결 매핑 (P1~P5)

· 톤&매너: 블랙 기반 다크 럭셔리 + Playfair Display(세리프)로 헤리티지의 격,
  Noto Sans KR로 본문 가독성 확보 → '고전적 권위 + 현대적 명료함'.
· 프리로더(#preloader): 카운트·바가 차오르며 브랜드명 노출 → 로딩을 '개막'으로 전환. (P5 해결)
· 히어로(#hero): 실사 카메라 이미지 + <canvas> WebGL glow, 클립된 2행 타이틀 리빌,
  SCROLL 인디케이터로 다음 행동 유도. (G1)
· About(#about): 1941 카운트업 + 대형 세리프 "역사" + 패럴랙스 이미지 그리드로
  헤리티지를 첫 서사 블록으로 승격. (P1 해결)
· 키네틱 마퀴: "Create to Inspire — Light defines legacy — Since 1941" 정/역방향 흐름으로
  브랜드 슬로건을 리듬으로 각인. (P4)
· Products(#products): 가로 핀 쇼케이스 카드(01 X / 02 V / 03 H / 04 Lens) →
  개별 제품이 아니라 '시스템의 순서'로 읽히게 재구조화. (P2 해결)
· Inspiration(#inspiration): 피처 스토리 + 작가·장비 맥락 캡션으로 흩어진 작례에 서사를 부여. (P3)
· Ambassadors: 전문가 인용구·포트레이트로 사회적 증거(신뢰) 제공.
· Discover(#discover): data-span 기반 비대칭 매트릭스 갤러리 + 라이트박스 확대로
  결과물을 '하나의 갤러리'에 응집해 주인공으로 전시. (P3 해결)
· 접근성: skip-link(#about), 섹션별 aria-labelledby, 의미 있는 alt, 커서/링/마퀴는 aria-hidden.

### 기대 효과 (Expected Outcome)

· 진입 즉시 '다크 럭셔리 헤리티지'가 체감되어 초기 이탈이 줄고 브랜드 격 인지가 상승. (P1·P5)
· 제품이 '시스템'으로 조망되어 전문가의 라인업 이해 시간이 단축된다. (P2)
· 작례가 주인공으로 전시되어 '나도 저런 이미지를' 동경이 커지고 구매 동기가 강화된다. (P3)
· 프리로더·WebGL·클립 리빌이 '정밀·빛'이라는 브랜드 속성을 정서적으로 각인. (P4)
· skip-link·aria·alt로 화려함과 접근성을 동시에 만족 → 폭넓은 사용자 포용.

## 2. 추가 결함 보강 — 반영 (HP1~HP4)

[② 개선안 §3]의 2차 개선(구현 레벨 결함 보강)을 실제로 반영하고 Playwright로 검증한 기록이다.

### HP1 — 모바일 내비게이션 드로어

· 발견한 문제: ≤720px에서 nav 메뉴를 대체 없이 display:none → 모바일 섹션 이동 수단 전무.
· 개선 목표: 모바일에서 About·Products·Inspiration·Discover 탐색 회복.
· 디자인 해결: 우상단 햄버거 → dark luxury 풀스크린 드로어(Playfair 헤딩 + 부제).
  aria-expanded/controls, 포커스 트랩, ESC, 스크롤 락(hb_lenis), 포커스 복귀.
  reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.
· 검증: 버거 노출·드로어 열림·focusInside=true·ESC 닫힘 통과.

### HP2 — READ MORE 죽은 링크 처리

· 발견한 문제: 다수 "READ MORE"(.link-more)가 href="#" → 클릭 시 페이지 최상단 점프.
· 개선 목표: 오작동 제거, 상세 페이지 부재를 정직하게 처리.
· 디자인 해결: href="#"인 link-more 6개에 aria-disabled="true" + 클릭 무력화(preventDefault).
· 기대 효과: 잘못된 상호작용 제거, 혼란 방지.
· 검증: link-more 6개 중 6개 aria-disabled 적용 확인.

### HP3 — 라이트박스 포커스 트랩 보강

· 발견한 문제: 라이트박스가 열려도 Tab 시 포커스가 뒤 페이지로 이탈(모달 트랩 미비).
· 개선 목표: 모달 열림 동안 포커스를 모달 내부로 가둔다.
· 디자인 해결: open 상태에서 Tab을 닫기 버튼으로 고정.
· 기대 효과: 키보드/스크린리더 모달 조작 일관성.
· 검증: 라이트박스 열림→Tab 시 포커스 닫기 버튼 유지, ESC 닫힘 확인.

### HP4 — 데스크톱 nav 현재 섹션 활성 표시

· 발견한 문제: nav 링크에 현재 섹션 표시가 없어 wayfinding 약함.
· 개선 목표: 현재 위치를 nav에 표시.
· 디자인 해결: 스크롤 시 활성 섹션 링크 aria-current="location" + 밑줄 유지.
· 기대 효과: 데스크톱 wayfinding 향상.
· 검증: 스크롤 시 활성 nav(#discover 등) 정확히 갱신 확인.

### 종합 검증(실측)

· 콘솔/페이지 오류 0 · 실패 네트워크 요청 0 · 가로 오버플로우 0 (데스크톱·모바일)
· 데스크톱: 활성 nav·라이트박스 트랩·READ MORE 무력화 정상 / 모바일: 드로어·포커스·ESC 정상
· 컨셉(dark luxury·프리로더·WebGL 히어로·가로 핀 쇼케이스·라이트박스) 보존, 신규 라이브러리 0.
· 참고: 상단 스크롤 진행바는 모바일에도 이미 노출되어 별도 게이지 불필요.

남은 사항: 제품 상세 페이지 확보 시 READ MORE를 실제 링크로 승격(placeholder 처리).

[검증 메모] 리디자인은 콘텐츠·IA·인터랙션 구조를 코드 레벨에서 확인해 작성했다.

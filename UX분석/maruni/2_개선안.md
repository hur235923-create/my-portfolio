# MARUNI 리디자인 — UX 개선 설계안 (Improvement Design Spec)

**성격: 결함 보강(컨셉 보존) · 대상: 나의-포트폴리오/works/maruni/**  
**근거: 03_Maruni_UX분석.txt · 설계: 허강범 · 2026-07-04**  

컨셉(quiet luxury · 뮤지엄 에디토리얼 · SplitType 리빌 · GSAP Flip 라이트박스)은
보존하고, 코드에서 확인된 실재 결함만 보강한다.

우선순위
  MP1 🔴 모바일 내비게이션 (nav__menu 소멸 대체)
  MP2 🔴 라이트박스 키보드 접근 + 포커스 트랩
  MP3 🟡 데스크톱 nav 현재 섹션 활성 표시
  MP4 🟡 푸터 SNS 죽은 링크 처리 + 뉴스레터 피드백

## MP1 — 모바일 내비게이션 🔴

· 문제: ≤920px에서 nav 메뉴를 대체 없이 display:none → 모바일 섹션 이동 수단 전무.
  근거: styles.css:253–254 @media(max-width:920px){ .nav__menu{ display:none } }.
· 개선 목표: 모바일에서 Collection·Craft·Designers 탐색 회복.
· 디자인 해결: 우상단 햄버거 → 웜 아이보리 드로어(Cormorant 헤딩 + 부제).
  aria-expanded/controls, 포커스 트랩, ESC, 스크롤 락(mr_lenis), 포커스 복귀.
  reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.

## MP2 — 라이트박스 키보드 접근 + 포커스 트랩 🔴

· 문제: Flip 라이트박스가 .piece__media img '마우스 클릭'으로만 열려 키보드/스크린리더로는
  가구 확대(핵심 인터랙션)에 접근 불가. 또한 모달 열림 시 Tab 포커스 트랩 없음.
  근거: script.js:169–173 (img click만, tabindex/role/keydown 없음, Tab 트랩 없음).
· 개선 목표: 키보드로 확대 열기 + 모달 포커스 가두기.
· 디자인 해결: piece 이미지에 tabindex=0 · role=button · aria-label(가구명) 부여,
  Enter/Space로 열기. 라이트박스 open 상태에서 Tab을 닫기 버튼으로 고정.
· 기대 효과: 소장 가치의 핵심 경험(디테일 확대)을 모든 사용자에게 균등 제공.

## MP3 — 데스크톱 nav 현재 섹션 활성 표시 🟡

· 문제: nav 링크에 현재 섹션 표시가 없고 진행 지표도 없어 wayfinding이 약함.
· 개선 목표: 현재 위치를 nav에 표시.
· 디자인 해결: 스크롤 시 활성 섹션 링크 aria-current="location" + 밑줄 유지
  (quiet luxury 유지 위해 진행 바는 추가하지 않음).
· 기대 효과: 데스크톱 wayfinding 향상.

## MP4 — 푸터 SNS 죽은 링크 + 뉴스레터 피드백 🟡

· 문제: ① SNS(Instagram/Pinterest/YouTube)가 href="#" → 클릭 시 상단 점프.
  ② 뉴스레터 폼(onsubmit=return false)이 제출 시 아무 피드백이 없어 동작 여부 불명.
· 개선 목표: 오작동 제거 + 폼 상호작용에 명확한 응답.
· 디자인 해결: ① SNS에 aria-disabled + 클릭 무력화. ② 제출 시 aria-live 메시지로
  접수/입력 안내 표시(절제된 캡션 톤).
· 기대 효과: 잘못된 상호작용 제거, 폼 신뢰·피드백 확보.

검증 체크리스트: 콘솔/네트워크 0오류, 가로 오버플로우 0, 드로어 키보드/트랩/ESC,
라이트박스 키보드 열기·트랩, nav aria-current, 뉴스레터 피드백, reduced-motion,
데스크톱+모바일 스크린샷.
범위 밖: 실제 SNS/뉴스레터 백엔드 연동(placeholder 처리).

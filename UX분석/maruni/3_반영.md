# MARUNI 리디자인 — UX 케이스 스터디 ③ 반영 (Implementation)

**"Minimal in form, deep in soul" / Wood Furniture · Quiet Luxury**
**변경 파일: works/maruni/ index.html · styles.css · script.js**
**검증: 로컬 서버 + Playwright (데스크톱 1440 / 모바일 390)**
**작성: 허강범 · 2026**

[② 개선안](2_개선안.md)의 설계를 실제 화면·코드로 구현한 결과다.
[① 분석](1_분석.md)의 문제 **P1~P5**가 화면에서 어떻게 해결됐는지(§1)와,
완성 후 추가로 잡은 결함 보강의 반영(§2)을 함께 기록한다.

## 1. 리디자인 구현 — 문제 해결 매핑 (P1~P5)

· 톤&매너: 웜 뉴트럴 팔레트 + Cormorant Garamond/Noto Serif KR 세리프로 뮤지엄 에디토리얼
  분위기. 큰 여백과 얇은 헤어라인으로 quiet luxury의 정적을 조형화.
· Hero(#hero): 정원이 보이는 대형 우드 공간 이미지 + veil + "Minimal in form, deep in soul"
  라인 리빌로 톤을 즉시 각인. Scroll 앵커로 자연스러운 진입.
· Manifesto(#manifesto): 'industrialised craft' 선언을 첫 서사 블록으로 승격, reveal-line으로
  한 줄씩 차오르게 → 철학을 제품보다 먼저 배치. (P1 해결)
· Collection(#collection): Roundish/Lightwood/Hiroshima를 좌/우 교차 배치, 각 가구에 형태·감성
  카피 + 디자이너 크레딧(Fukasawa/Morrison) 부여. (P2·P4 해결)
· 라이트박스(#lightbox, GSAP Flip): 가구 이미지를 클릭하면 썸네일→전체화면으로 이어지는
  공유요소 전환. 곡면·나뭇결을 맥락 유지한 채 확대. (P3·P5 해결)
· Craft(#craft): 장인의 손 이미지 + facts(1928 / 부품 10 / 5-axis CNC+Hand)로 만듦새를
  구체 데이터로 증거화. (P4)
· Designers(#designers): 모리슨 포트레이트 + 'Super Normal' 서사 + "Minimal in form, deep in
  soul" 인용으로 소장 가치의 근거(사람) 제공.
· Footer: "Where craftsmanship meets modern design" CTA + 뉴스레터 폼(라벨 visually 제공).

### 기대 효과 (Expected Outcome)

· 에디토리얼 서사로 사용자가 '비교'에서 '음미' 모드로 전환되어 브랜드 격을 체감한다. (P2)
· 철학·만듦새·디자이너가 제품과 한 흐름으로 통합되어 소장 가치가 설득력 있게 전달된다. (P1·P4)
· Flip 공유요소 확대로 곡면·결 디테일을 맥락 유지한 채 감상 → 구매 확신이 강화된다. (P3·P5)
· SplitType 리빌의 '느림'이 quiet luxury의 정서를 시간 축으로 각인한다.
· 세리프 감성과 본문 가독·폼 접근성의 병행으로 미감과 사용성을 동시에 만족.

## 2. 추가 결함 보강 — 반영 (MP1~MP4, 구현 레벨)

[② 개선안 §3]의 2차 개선(구현 레벨 결함 보강)을 실제로 반영·검증한 기록이다.
각 변경은 [발견한 문제 → 개선 목표 → 디자인 해결 방법 → 기대 효과 → 검증]으로 기술한다.

## MP1 — 모바일 내비게이션 드로어

· 발견한 문제: ≤920px에서 nav 메뉴를 대체 없이 display:none → 모바일 섹션 이동 수단 전무.
· 개선 목표: 모바일에서 Collection·Craft·Designers 탐색 회복.
· 디자인 해결: 우상단 햄버거 → 웜 아이보리 드로어(Cormorant 헤딩 + 부제). aria-expanded/controls,
  포커스 트랩, ESC, 스크롤 락(mr_lenis), 포커스 복귀. reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.
· 검증: 버거 노출·드로어 열림·focusInside=true·ESC 닫힘 통과.

## MP2 — 라이트박스 키보드 접근 + 포커스 트랩

· 발견한 문제: Flip 라이트박스가 마우스 클릭으로만 열려 키보드/스크린리더로는 가구 확대(핵심
  인터랙션) 접근 불가. 모달 Tab 트랩도 없음.
· 개선 목표: 키보드로 확대 열기 + 모달 포커스 가두기.
· 디자인 해결: piece 이미지에 tabindex=0 · role=button · aria-label(가구명), Enter/Space로 열기.
  open 상태에서 Tab을 닫기 버튼으로 고정.
· 기대 효과: 소장 가치의 핵심 경험을 모든 사용자에게 균등 제공.
· 검증: tabindex/role/aria-label 부여, Enter 열기·Tab 트랩·ESC 닫힘 확인
  (예: "Roundish 확대 보기").

## MP3 — 데스크톱 nav 현재 섹션 활성 표시

· 발견한 문제: nav 활성 표시·진행 지표 부재로 wayfinding 약함.
· 개선 목표: 현재 위치를 nav에 표시.
· 디자인 해결: 스크롤 시 활성 섹션 링크 aria-current="location" + 밑줄 유지
  (quiet luxury 유지 위해 진행 바는 미추가).
· 기대 효과: 데스크톱 wayfinding 향상.
· 검증: 스크롤 시 활성 nav(#designers 등) 정확히 갱신 확인.

## MP4 — 푸터 SNS 죽은 링크 + 뉴스레터 피드백

· 발견한 문제: SNS(Instagram/Pinterest/YouTube) href="#" 상단 점프 / 뉴스레터 제출 시 무반응.
· 개선 목표: 오작동 제거 + 폼 상호작용에 명확한 응답.
· 디자인 해결: SNS aria-disabled + 클릭 무력화 + dim. 제출 시 aria-live 메시지로 접수/입력 안내.
· 기대 효과: 잘못된 상호작용 제거, 폼 피드백 확보.
· 검증: SNS 3개 aria-disabled, 입력 있음→"구독 신청이 접수되었습니다", 없음→"이메일 주소를
  입력해 주세요" 분기 확인.

## 종합 검증(실측)

· 콘솔/페이지 오류 0 · 실패 네트워크 요청 0 · 가로 오버플로우 0 (데스크톱·모바일)
· 데스크톱: 활성 nav·라이트박스 키보드/트랩·뉴스레터 피드백 정상 / 모바일: 드로어·포커스·ESC 정상
· 컨셉(quiet luxury·뮤지엄 에디토리얼·SplitType·Flip 라이트박스) 보존, 신규 라이브러리 0.

남은 사항: 실제 SNS/뉴스레터 백엔드 연동(placeholder 처리).

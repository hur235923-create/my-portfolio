# YAMAHA 리디자인 — UX 케이스 스터디

**"Make Waves" / Sound Brand · 3D Interactive**  
**분석: 허강범 (Heo Gangbeom) · 2026**  

완성된 리디자인을 UX 관점에서 역설계하여, 무엇이 문제였는지 → 왜 문제인지 →
어떻게 해결했는지의 논리로 정리한다. '기존'은 대형 종합 브랜드(악기·음향·산업기계)
사이트의 통상적 패턴을 기준으로 한다.

## 1. 서비스 이해 (Service Understanding)

· 브랜드 목적
  - 1887년 창립. 악기·음향기기·산업기계/부품을 아우르지만 모든 사업의 중심은 '소리와 음악'.
  - 브랜드 슬로건 "Make Waves" — 소리로 세상에 파동(영향)을 일으킨다는 정체성.

· 핵심 가치
  - 한 세기 넘는 장인정신 · 소리에 대한 전문성 · 사람과 사람을 잇는 음악 · 전 세계인의 웰빙.

· 주요 사용자
  - (A) 연주자/음악 애호가: 악기 라인업과 브랜드 철학에 감정적으로 공감하려는 사람.
  - (B) 브랜드 탐색층: '야마하는 어떤 회사인가'를 이해하려는 일반/잠재 고객.
  - (C) 채용담당자/동료: 3D·모션 구현력과 브랜드 재해석 역량을 평가.

· 방문 목적
  - 방대한 사업군에 압도되지 않고 '소리 브랜드'라는 하나의 정체성을 체감.
  - 악기(제품)의 감성과 브랜드가 지켜온 신뢰(사회적 증거)를 확인.

## 2. 사용자 목표 분석 (User Goals)

· 가장 먼저 찾는 정보
  - "이 브랜드가 지향하는 소리/철학" → "대표 악기군의 감성" → "믿을 만한가(신뢰 지표)".

· 주요 행동 흐름
  1) Hero "Make Waves"에서 브랜드 정서 감지
  2) Resonance(About)에서 '비움이 만드는 울림' 철학 이해
  3) Instruments에서 Piano/Strings/Guitar의 감성 확인
  4) Voices에서 수치·연주자 인용으로 신뢰 형성
  5) Legacy에서 존재 이유(웰빙)로 브랜드 각인 → 잔상 형성

· 사용자의 진짜 니즈
  - 산업기계까지 포함한 '거대 기업'의 복잡함이 아니라, 감성적으로 정돈된 '소리의 세계'.
  - 제품 스펙 나열이 아니라 '이 브랜드가 소리를 대하는 태도'.

## 3. 기존 UX 분석 (Baseline UX Diagnosis)

· 정보구조(IA)
  - 다각화된 사업군(악기/음향/산업)이 병렬 메뉴로 나열되어 '소리 브랜드' 정체성이 희석.
  - 제품 카테고리가 지나치게 세분화되어 브랜드 서사가 파편화됨.

· 내비게이션
  - 글로벌 대기업 특성상 국가/사업/제품의 다층 구조 → 탐색 경로가 길고 정체성 인지 지연.

· 시각적 계층
  - 제품 그리드가 균질하게 반복되어 '어떤 악기의 감성을 먼저 느낄지' 위계가 약함.

· 콘텐츠
  - 스펙·라인업 중심. '소리를 어떻게 대하는가'라는 정서적 메시지가 부족.

· 인터랙션
  - 정적 카드/캐러셀 위주. '파동(wave)·울림'이라는 브랜드 핵심 은유가 움직임으로 없음.

· 접근성
  - 3D/모션을 넣을 경우 스크린리더·키보드·모션 민감 사용자 배려가 누락되기 쉬움.

· 모바일 사용성
  - 무거운 3D·대형 이미지의 성능 부담, 우측 고정 내비의 터치 여백 문제.

## 4. 문제 발견 (Pain Points)

P1. 방대한 사업군에 정체성이 희석되어 '소리 브랜드'라는 핵심 메시지가 흐려진다.
P2. 제품이 균질 그리드로 나열되어 각 악기의 감성적 차이가 위계로 전달되지 않는다.
P3. '파동·울림'이라는 브랜드 핵심 은유를 체감할 인터랙션 장치가 없다.
P4. 신뢰 근거(사회적 증거)가 흩어져 있어 '믿을 만한가'가 한 흐름으로 설득되지 않는다.
P5. 긴 페이지에서 '내가 지금 어디쯤인가'의 탐색 지표가 약하다.

## 5. 원인 분석 (Why)

P1 ← 사업 다각화를 IA에 그대로 반영해 '소리'라는 상위 개념으로 통합하지 못함.
P2 ← 제품을 카탈로그 단위로 다루고 악기별 서사·감성 위계를 설계하지 않음.
P3 ← 모션을 장식으로만 인식. 브랜드 은유(wave)를 관통하는 시각 장치가 부재.
P4 ← 수치·인용·연혁이 각기 다른 페이지에 분산되어 신뢰가 누적되지 않음.
P5 ← 롱스크롤 단일 서사에 맞는 '진행 + 위치' 내비 패턴을 갖추지 않음.

## 6. UX 문제 정의 (Problem Statement)

· 사용자는 "야마하가 소리를 대하는 태도와 대표 악기의 감성"을 느끼려 하지만,
  방대한 사업군과 균질한 제품 나열 때문에 '소리 브랜드'라는 핵심 정체성에 도달하지 못한다.
· 사용자는 긴 페이지에서 "지금 어디를 보고 있는가"를 알고 싶지만, 위치·진행 지표가 약해
  탐색의 확신이 떨어진다.

## 7. 프로젝트 목표 (Goal)

G1. 복잡한 사업군을 걷어내고 'Make Waves — 소리 브랜드' 하나의 정체성으로 집중시킨다.
G2. 'wave/울림'이라는 은유를 페이지 전체를 관통하는 시각 장치로 구현한다.
G3. 악기를 감성 위계가 있는 서사로 재편하고, 신뢰 근거를 한 흐름으로 누적시킨다.
G4. 롱스크롤에 맞는 '진행 + 위치' 내비를 제공하되 3D 성능·접근성을 지킨다.

## 8. UX 전략 (Strategy)

S1. 정체성 집중 IA: Resonance(철학) → Instruments(제품) → Voices(신뢰) → Legacy(존재이유)의
    4막 단일 서사로 재편, 산업기계 등 비핵심 사업은 서사에서 의도적으로 배제.
S2. 관통하는 은유: 절차적 MatCap 크롬 '사운드 링'(Three.js)이 스크롤을 따라 떠다니며
    페이지 전체의 '스파인(척추)' 역할 → 브랜드 은유(파동)를 상시 체감(G2).
S3. 감성 위계형 제품: Instruments를 좌/우 교차 배치(data-side)로 리듬을 주고,
    Piano/Strings/Guitar마다 '소리를 대하는 문장'을 부여해 감성 차이를 위계화.
S4. 신뢰의 누적: Voices에서 카운트업 수치(1887/80+/138) + 연주자 인용을 한 섹션에 집약해
    사회적 증거를 하나의 클라이맥스로 설계.
S5. 탐색 지표: 우측 '주파수 눈금(ruler)' = 내비 + 진행 표시로 위치·진척을 동시에 제공(G4).
S6. 접근성·성능 병행: skip-link, 3D 캔버스 aria-hidden, 시맨틱 섹션, 이미지 대체텍스트.

## 9. UI 적용 (Design Solution — 실제 구현 매핑)

· 톤&매너: 여백을 크게 둔 미니멀 타이포(Noto Sans KR 100–700 웨이트 폭)로 '비움 = 울림'
  컨셉을 조형화. 극단적 라이트/헤어라인 웨이트로 고요함을, 굵은 헤딩으로 존재감을 교차.
· 사운드 링(#ring, <canvas>): 페이지 전 구간을 관통하는 3D 크롬 오브젝트로 'wave' 은유를
  물리적으로 구현(P3 해결, G2). aria-hidden으로 접근성 보호.
· Hero(#hero): "Make / Waves" 들여쓰기 2행 타이틀 + '스크롤하여 울림을 따라가세요' 유도.
· Resonance(#resonance): "비움이 만드는 울림" 대형 헤딩 + 세 사업을 '소리 중심'으로 통합 서술
  → 정체성 집중(P1 해결, G1). 피아노 건반 흑백 플레이트로 정서 각인.
· Instruments(#instruments): 좌/우 교차 아이템(01 Keys/02 Strings/03 Acoustic)으로
  악기별 감성 위계와 리듬 부여(P2 해결).
· Voices(#voices): 카운트업 figures(설립연도·국가수·연수) + 연주자 blockquote 3종으로
  신뢰를 한 흐름에 집약(P4 해결).
· Legacy(#legacy): "Well-being of people around the world" 초대형 헤딩 + Purpose/Philosophy
  2열로 브랜드 존재 이유를 잔상으로 남김.
· Ruler(우측 aside): rulerFill 진행 바 + 01–04 섹션 앵커로 위치·진척 제공(P5 해결, G4).
· 접근성: skip-link(#resonance), foot 소셜/주소 시맨틱, 캔버스·아이콘 aria-hidden.

## 10. 기대 효과 (Expected Outcome)

· 방대한 사업군이 걷힌 4막 서사로 '소리 브랜드' 정체성이 명확히 각인된다.
· 관통하는 사운드 링이 'Make Waves' 은유를 상시 체감시켜 브랜드 기억도를 높인다.
· 악기별 감성 위계로 제품이 '카탈로그'가 아닌 '이야기'로 읽혀 몰입이 상승한다.
· Voices의 수치·인용 집약으로 신뢰가 한 번에 누적되어 설득력이 강화된다.
· 우측 ruler로 롱스크롤에서도 위치·진척이 명확해 탐색 확신이 커진다.

[검증 메모] IA·콘텐츠·인터랙션 구조를 코드 레벨에서 확인해 작성했다.
Three.js 사운드 링의 실제 60fps 성능·모바일 발열/로딩과 우측 ruler의 터치 타깃은
브라우저 실측(데스크톱+모바일 2해상도)으로 별도 확인 권장.

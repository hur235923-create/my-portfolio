## 1. 리디자인 구현 — 문제 해결 매핑 (P1~P5)

· 톤&매너: 여백을 크게 둔 미니멀 타이포(Noto Sans KR 100–700 웨이트 폭)로 '비움 = 울림'
  컨셉을 조형화. 극단적 라이트/헤어라인 웨이트로 고요함을, 굵은 헤딩으로 존재감을 교차.
· 사운드 링(#ring, <canvas>): 페이지 전 구간을 관통하는 3D 크롬 오브젝트로 'wave' 은유를
  물리적으로 구현. (P3 해결) aria-hidden으로 접근성 보호.
· Hero(#hero): "Make / Waves" 들여쓰기 2행 타이틀 + '스크롤하여 울림을 따라가세요' 유도.
· Resonance(#resonance): "비움이 만드는 울림" 대형 헤딩 + 세 사업을 '소리 중심'으로 통합 서술
  → 정체성 집중. (P1 해결) 피아노 건반 흑백 플레이트로 정서 각인.
· Instruments(#instruments): 좌/우 교차 아이템(01 Keys/02 Strings/03 Acoustic)으로
  악기별 감성 위계와 리듬 부여. (P2 해결)
· Voices(#voices): 카운트업 figures(설립연도·국가수·연수) + 연주자 blockquote 3종으로
  신뢰를 한 흐름에 집약. (P4 해결)
· Legacy(#legacy): "Well-being of people around the world" 초대형 헤딩 + Purpose/Philosophy
  2열로 브랜드 존재 이유를 잔상으로 남김.
· Ruler(우측 aside): rulerFill 진행 바 + 01–04 섹션 앵커로 위치·진척 제공. (P5 해결)
· 접근성: skip-link(#resonance), foot 소셜/주소 시맨틱, 캔버스·아이콘 aria-hidden.

### 기대 효과 (Expected Outcome)

· 방대한 사업군이 걷힌 4막 서사로 '소리 브랜드' 정체성이 명확히 각인된다. (P1)
· 관통하는 사운드 링이 'Make Waves' 은유를 상시 체감시켜 브랜드 기억도를 높인다. (P3)
· 악기별 감성 위계로 제품이 '카탈로그'가 아닌 '이야기'로 읽혀 몰입이 상승한다. (P2)
· Voices의 수치·인용 집약으로 신뢰가 한 번에 누적되어 설득력이 강화된다. (P4)
· 우측 ruler로 롱스크롤에서도 위치·진척이 명확해 탐색 확신이 커진다. (P5)

## 2. 추가 결함 보강 — 반영 (YP1~YP4, 구현 레벨)

[② 개선안 §3]의 2차 개선(구현 레벨 결함 보강)을 실제로 반영·검증한 기록이다.
각 변경은 [발견한 문제 → 개선 목표 → 디자인 해결 방법 → 기대 효과 → 검증]으로 기술한다.

## YP1 — 모바일 내비게이션 드로어

· 발견한 문제: 우측 ruler(유일한 내비+진행)가 ≤900px에서 사라져 모바일 섹션 이동 수단 전무.
· 개선 목표: 모바일에서 Resonance·Instruments·Voices·Legacy 탐색 회복.
· 디자인 해결: 우상단 햄버거 → 페이퍼 톤 풀스크린 드로어(섹션+부제). aria-expanded/controls,
  포커스 트랩, ESC, 스크롤 락(ym_lenis), 포커스 복귀. reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.
· 검증: 버거 노출·드로어 열림·focusInside=true·ESC 닫힘 통과.

## YP2 — 모바일 진행 게이지 (ruler 대체)

· 발견한 문제: ruler 소멸로 진행/위치감(rulerFill)이 모바일에서 사라짐.
· 개선 목표: 모바일에서도 진행·위치감 유지.
· 디자인 해결: 최상단 얇은 violet 진행 바. 기존 rulerFill 계산 재사용, scaleX만 반영.
· 기대 효과: 롱스크롤 wayfinding 회복.
· 검증: 모바일 게이지 노출 확인.

## YP3 — ruler 키보드 접근성

· 발견한 문제: ruler 라벨(span)이 hover/active에서만 노출, :focus에서 숨겨져 키보드 사용자가
  섹션을 알 수 없음.
· 개선 목표: 키보드 포커스 시 라벨 노출 + 현재 섹션 표시.
· 디자인 해결: :focus-visible span 노출, 활성 섹션에 aria-current="location"(+라벨 노출).
· 기대 효과: 키보드·스크린리더 wayfinding 향상.
· 검증: #instruments/#voices/#legacy 스크롤 시 각 항목 aria-current 정확히 갱신 확인.

## YP4 — 푸터 SNS 죽은 링크

· 발견한 문제: SNS(Instagram/YouTube/Facebook)가 href="#" → 클릭 시 상단 점프.
· 개선 목표: 오작동 제거, 미완 상태 명시.
· 디자인 해결: aria-disabled="true" + 클릭 무력화(preventDefault) + 시각적 dim.
· 기대 효과: 잘못된 상호작용 제거.

## 종합 검증(실측)

· 콘솔/페이지 오류 0 · 실패 네트워크 요청 0 · 가로 오버플로우 0 (데스크톱·모바일)
· 데스크톱: 활성 ruler 정상 / 모바일: 드로어·포커스 트랩·ESC·게이지 정상 · 3D 사운드 링 렌더 정상
· 컨셉(Make Waves · 라이트 갤러리 · 사운드 링) 보존, 결함만 국소 보강. 신규 라이브러리 0.

남은 사항: 실제 SNS URL 확보 시 즉시 연결(placeholder 처리).

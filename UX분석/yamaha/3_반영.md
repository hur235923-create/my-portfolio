## YP1 — 모바일 내비게이션 드로어

· 발견한 문제: 우측 ruler(유일한 내비+진행)가 ≤900px에서 사라져 모바일 섹션 이동 수단 전무.
· 개선 목표: 모바일에서 Resonance·Instruments·Voices·Legacy 탐색 회복.
· 디자인 해결: 우상단 햄버거 → 페이퍼 톤 풀스크린 드로어(섹션+부제). aria-expanded/controls,
  포커스 트랩, ESC, 스크롤 락(ym_lenis), 포커스 복귀. reduced-motion 시 트랜지션 제거.
· 기대 효과: 모바일 탐색 경로 회복.
· 검증: 버거 노출·드로어 열림·focusInside=true·ESC 닫힘 통과.

## YP2 — 모바일 진행 게이지 (ruler 대체)

· 발견한 문제: ruler 소멸로 진행/위치감(rulerFill)이 모바일에서 사라짐.
· 개선 목표: 모바일에서도 진행·위치감 유지.
· 디자인 해결: 최상단 얇은 violet 진행 바. 기존 rulerFill 계산 재사용, scaleX만 반영.
· 기대 효과: 롱스크롤 wayfinding 회복.
· 검증: 모바일 게이지 노출 확인.

## YP3 — ruler 키보드 접근성

· 발견한 문제: ruler 라벨(span)이 hover/active에서만 노출, :focus에서 숨겨져 키보드 사용자가
  섹션을 알 수 없음.
· 개선 목표: 키보드 포커스 시 라벨 노출 + 현재 섹션 표시.
· 디자인 해결: :focus-visible span 노출, 활성 섹션에 aria-current="location"(+라벨 노출).
· 기대 효과: 키보드·스크린리더 wayfinding 향상.
· 검증: #instruments/#voices/#legacy 스크롤 시 각 항목 aria-current 정확히 갱신 확인.

## YP4 — 푸터 SNS 죽은 링크

· 발견한 문제: SNS(Instagram/YouTube/Facebook)가 href="#" → 클릭 시 상단 점프.
· 개선 목표: 오작동 제거, 미완 상태 명시.
· 디자인 해결: aria-disabled="true" + 클릭 무력화(preventDefault) + 시각적 dim.
· 기대 효과: 잘못된 상호작용 제거.

# YAMAHA 리디자인 — UX 개선 설계안 (Improvement Design Spec)

**성격: 결함 보강(컨셉 보존) · 대상: 나의-포트폴리오/works/Yamaha/**  
**근거: 02_Yamaha_UX분석.txt · 설계: 허강범 · 2026-07-04**  

컨셉(Make Waves · 라이트 갤러리 · 3D 사운드 링 · 우측 ruler)은 보존하고,
코드에서 확인된 실재 결함만 보강한다.

우선순위
  YP1 🔴 모바일 내비게이션 (ruler 소멸 대체)
  YP2 🔴 모바일 진행 게이지 (wayfinding)
  YP3 🟡 ruler 키보드 접근성
  YP4 🟡 푸터 SNS 죽은 링크(#) 처리

## YP1 — 모바일 내비게이션 🔴

· 문제: 우측 ruler가 유일한 내비게이션 겸 진행 표시인데 ≤900px에서 통째로 사라진다.
  근거: styles.css:229–230 @media(max-width:900px){ .ruler{ display:none } }.
  상단 헤더(.top)에는 브랜드+태그만 있어 모바일에 섹션 이동 수단이 전무.
· 개선 목표: 모바일에서도 4개 섹션(Resonance·Instruments·Voices·Legacy) 탐색 회복.
· 디자인 해결: 우상단 햄버거 → 페이퍼 톤 풀스크린 드로어(섹션+부제).
  aria-expanded/controls, 포커스 트랩, ESC, 스크롤 락(ym_lenis.stop/start), 포커스 복귀.
  reduced-motion 시 트랜지션 제거. 라이트 갤러리 톤(--paper/--violet) 유지.
· 기대 효과: 모바일 탐색 경로 회복 → 위치 이동·태스크 완결.

## YP2 — 모바일 진행 게이지 🔴

· 문제: ruler 소멸로 스크롤 진행/위치감(rulerFill)이 모바일에서 사라진다(분석 P5 위치감).
· 개선 목표: 모바일에서도 진행·위치감 유지.
· 디자인 해결: 최상단 얇은 진행 바(violet). 기존 rulerFill 스크롤 계산을 재사용해 scaleX만 반영.
· 기대 효과: 롱스크롤 wayfinding 회복.

## YP3 — ruler 키보드 접근성 🟡

· 문제: ruler 링크 라벨(span)이 hover/active에서만 노출되고 :focus에서는 숨겨져,
  키보드 사용자가 포커스 시 어떤 섹션인지 시각적으로 알 수 없다.
  근거: styles.css:89–90 (span opacity:0, hover/is-active만 노출).
· 개선 목표: 키보드 포커스 시에도 라벨 노출 + 현재 섹션 표시.
· 디자인 해결: .ruler__list a:focus-visible span 노출, 활성 섹션에 aria-current="location".
· 기대 효과: 키보드·스크린리더 사용자 wayfinding 향상.

## YP4 — 푸터 SNS 죽은 링크 🟡

· 문제: 푸터 SNS(Instagram/YouTube/Facebook)가 href="#" → 클릭 시 맨 위로 점프하는 죽은 링크.
  근거: index.html footer의 a href="#".
· 개선 목표: 오작동(상단 점프) 제거, 미완 상태 명시.
· 디자인 해결: aria-disabled="true" + 클릭 무력화(preventDefault). 외부 URL 확보 시 즉시 연결.
· 기대 효과: 잘못된 상호작용 제거, 신뢰 저하 방지.

검증 체크리스트: 콘솔/네트워크 0오류, 가로 오버플로우 0, 드로어 키보드/포커스 트랩/ESC,
모바일 게이지 연동, ruler focus 라벨, reduced-motion, 데스크톱+모바일 스크린샷.
범위 밖: 실제 SNS/문의 페이지 신설(placeholder 처리).

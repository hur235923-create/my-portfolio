# JW중외제약 케이스 스터디 페이지 — 설계 스펙

**작성: 허강범 · 2026-07-04**
**대상: 나의-포트폴리오/works/jw중외제약/casestudy.html (신설)**
**성격: 리디자인 결과물 뒤의 UX 사고 과정을 12단계 스토리텔링으로 노출하는 케이스 스터디 상세 페이지**

---

## 1. 목표 (Goal)

메인 포트폴리오(가로 갤러리)의 JW중외제약 패널은 지금 **결과물 링크만** 제공한다.
그 결과에 도달한 **UX 사고 과정(왜 이렇게 설계했는가)**을 세로 스크롤 케이스 스터디로
보여줘, 채용담당자가 "결과물 → 그 뒤의 과정"으로 자연스럽게 들어가도록 한다.

이번 스펙은 **JW중외제약 1건 파일럿**이다. 톤·구조를 확정한 뒤 나머지 작업물로 복제한다.

## 2. 범위 (Scope)

**In scope**
- `works/jw중외제약/casestudy.html` 신설 (세로 스크롤, '고요의 심해' 톤).
- 전용 CSS/JS (파일명 `casestudy.css` / `casestudy.js`, 또는 인라인 — 구현 시 확정).
- 메인 `index.html`의 JW중외제약 패널(panel-4)에 **"케이스 스터디 →" 진입 버튼 1개** 추가.
- 07/09 섹션용 결과물 스크린샷을 **로컬 서버 + Playwright로 직접 캡처**해 페이지에 사용.

**Out of scope**
- 나머지 3개 작업물(Hasselblad/Yamaha/Maruni)의 케이스 스터디 — 파일럿 확정 후 별도 진행.
- 결과물(`works/jw중외제약/index.html`) 자체의 재수정.
- 메인 갤러리 구조 변경(패널 추가·순서 변경). 버튼 1개만 추가.

## 3. 구조 & 동선 (Architecture)

```
메인 갤러리 (index.html)
  └ JW중외제약 패널(panel-4)
       ├ [라이브 보기 ↗]   → works/jw중외제약/index.html (기존 결과물, 유지)
       └ [케이스 스터디 →]  → works/jw중외제약/casestudy.html (신설) ← 이번 작업

casestudy.html (세로 스크롤, 심해 톤, 독립 실행)
  ├ 상단 고정 HUD: 세로/상단 진행 게이지(흐름 은유) + 섹션 카운터
  │                + "← 포트폴리오"(index.html) · "라이브 보기 ↗"(결과물)
  ├ 12개 세로 섹션 (§4)
  └ 하단: 다음 흐름(포트폴리오 복귀 / 라이브 사이트 CTA)
```

- 페이지는 **독립 실행**: GSAP + ScrollTrigger 세로 리빌 + 진행 게이지. 새 라이브러리 없음.
- 메인 index.html은 **패널에 버튼 1개만 추가**, 가로 갤러리 로직·구조는 불변.
- 접근성: skip-link, 섹션 `aria-labelledby`, 장식 `aria-hidden`, `prefers-reduced-motion` 대응.

## 5. 디자인 시스템 (결과물과 연속)

- **팔레트**: 심해 딥블루/블랙 배경(`--abyss`) + 시안 강조(`--cyan`) — 결과물 `:root` 토큰과
  동일 계열. 텍스트가 무거운 분석 구간(02~06,10,11)은 **반투명 유리 카드(살짝 밝힘)**로
  대비를 확보(가독성 규율). '보여주는' 구간(01,07,08,09)은 심해 톤 그대로 몰입.
- **타이포**: 제목 Noto Sans KR, 데이터·라벨 JetBrains Mono. 섹션 번호는 모노 세로 라벨.
- **모션**: 진행 게이지(흐름), 섹션 진입 클립 리빌, 카운트업 수치. 전부
  `prefers-reduced-motion:reduce`에서 축소/제거. GSAP + ScrollTrigger만 사용.
- **성능/반응형**: 데스크톱 / 태블릿(≤960) / 모바일(≤760) 3분기. 가로 오버플로우 0,
  backdrop-filter 남용 금지, 기존 이미지 자산 재사용.
- **이미지**: `works/jw중외제약/img/`(deep.jpg 등) 재사용 + 결과물 화면 캡처.
  부족 시 unsplash-downloader로 보강(장식 한정).

## 6. 검증 계획 (CLAUDE.md 워크플로우)

로컬 서버 기동 후 Playwright로 데스크톱 + 모바일 최소 2해상도에서 확인:

- [ ] 콘솔 오류 / 페이지 오류 / 실패 네트워크 요청 0건
- [ ] 가로 스크롤 오버플로우 없음 (12섹션 전체 + 모바일)
- [ ] 진행 게이지가 스크롤에 정확히 연동
- [ ] 섹션 진입 리빌 애니메이션 동작, `prefers-reduced-motion`에서 축소
- [ ] 메인 패널의 "케이스 스터디 →" 버튼 → casestudy.html 정상 진입
- [ ] casestudy HUD의 "← 포트폴리오" / "라이브 보기 ↗" 링크 정상
- [ ] 07/09 스크린샷이 실제 결과물과 일치, 깨진 이미지 0
- [ ] 데스크톱 + 모바일 스크린샷으로 레이아웃·디자인 의도 확인
- [ ] 검증용 임시 스크립트/스크린샷은 프로젝트에 남기지 않고 정리
